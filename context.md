# Project Context: Irion Architecture

## 🌐 Overview
The Irion Protocol is a decentralized finance platform built on **Algorand**, utilizing **Next.js 16** for the frontend and **MongoDB** for off-chain meta-data and transaction caching.

---

## 🏗️ Technical Stack

- **Blockchain**: Algorand (Testnet)
- **Node Provider**: AlgoNode (Public API)
- **Smart Contracts**: Puya TS (Algorand TypeScript)
- **Database**: MongoDB (Mongoose)
- **Frontend**: Next.js 16, Tailwind CSS, Shadcn UI
- **Wallet**: `@txnlab/use-wallet` (Pera, Defly, Lute)

---

## 🔌 Integration Wiring

### 1. Blockchain → MongoDB
- **Trigger**: Every contract call (mutated state) triggers a `POST` request to `/api/user/[address]/sync`.
- **Action**: The API fetches the latest state from the blockchain (Box storage) and updates the MongoDB `User` model.
- **Purpose**: Fast UI reads, transaction history indexing, and rich metadata storage not suitable for on-chain storage.

### 2. Smart Contract Interface
- **Clients**: Auto-generated by `algokit` based on ARC-56 app specs.
- **Hooks**: Centralized in `lib/hooks/useContractActions.ts` for atomic transaction management.

### 3. Account Configuration
- **Setup**: Users must opt-in to the **USDC Asset (10458941)** via the `SetupProtocol` component before borrowing or supplying.
- **Liquidity**: The `LendingPool` contract holds the liquidity, while the `BNPLCredit` contract handles loan logic and installments.
